# HRVU-202631204 - 高危漏洞修复

## 漏洞编号
**HRVU-202631204**

- **HRVU**: High-Risk Vulnerability Update（高危漏洞更新）
- **2026312**: 来源更新ID（2026年3月12日更新）
- **04**: 该更新后的第4个漏洞

## 漏洞描述

### 漏洞类型
路径遍历 / 敏感信息泄露

### 漏洞详情
`.keys` 目录虽然在目录列表中被过滤不显示，但攻击者可以通过手动构造URL直接访问该目录及其内容。

### 受影响路径
- `https://example.com/.keys`
- `https://example.com/.keys/server_private.pem`
- `https://example.com/.keys/server_public.pem`

### 风险等级
**高危**

### 影响范围
- 泄露服务器RSA私钥
- 泄露服务器公钥
- 可能导致Secure认证机制被破解
- 攻击者可伪造服务器身份

## 漏洞原因

在之前的修复中，仅在 `DirectoryController` 的 `ProcessDirectoriesAsync` 方法中过滤了 `.keys` 目录的显示，但没有在以下入口点进行访问控制：

1. `RepositoryController.Index` - 页面访问
2. `DirectoryController.GetFiles` - API目录列表
3. `FileController.PreviewFile` - 文件预览
4. `FileController.DownloadFile` - 文件下载
5. `UploadController.UploadFile` - 文件上传

## 修复方案

### 1. 添加系统路径检查方法
在 `ProtectionService` 中添加静态方法 `IsSystemPath`，用于检查路径是否包含系统目录：

```csharp
public static bool IsSystemPath(string path)
{
    if (string.IsNullOrEmpty(path))
        return false;
    
    var normalizedPath = path.Replace('\\', '/').Trim('/');
    var segments = normalizedPath.Split('/');
    
    var systemDirs = new[] { ".keys" };
    
    foreach (var segment in segments)
    {
        if (systemDirs.Any(sd => string.Equals(segment, sd, StringComparison.OrdinalIgnoreCase)))
        {
            return true;
        }
    }
    
    return false;
}
```

### 2. 在所有入口点添加检查

| 控制器 | 方法 | 检查位置 |
|--------|------|----------|
| RepositoryController | Index | 路径验证后 |
| DirectoryController | GetFiles | 路径验证后 |
| FileController | PreviewFile | 文件名检查后 |
| FileController | DownloadFile | 文件名检查后 |
| UploadController | UploadFile | 路径验证后 |

### 3. 统一返回404
所有系统路径访问请求统一返回 `404 Not Found`，不暴露路径是否存在的信息。

## 文件变更

### 修改文件
- `Services/ProtectionService.cs` - 添加 `IsSystemPath` 方法
- `Controllers/RepositoryController.cs` - 添加系统路径检查
- `Controllers/DirectoryController.cs` - 添加系统路径检查
- `Controllers/FileController.cs` - 添加系统路径检查（预览/下载）
- `Controllers/UploadController.cs` - 添加系统路径检查

## 验证方法

### 测试用例
1. 访问 `/.keys` - 应返回404
2. 访问 `/.keys/server_private.pem` - 应返回404
3. 访问 `/.keys/server_public.pem` - 应返回404
4. API请求 `/api/files?path=.keys` - 应返回404
5. 下载请求 `/api/download/.keys/server_private.pem` - 应返回404
6. 上传到 `/.keys/` - 应返回404

### 预期结果
所有测试用例均返回 `404 Not Found`，不泄露任何系统目录信息。
