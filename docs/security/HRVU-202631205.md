# HRVU-202631205 - Configuration Update

## Number
**HRVU-202631205**

- **HRVU**: High-Risk Vulnerability Update
- **2026312**: Source Update ID (March 12, 2026 update)
- **05**: Fifth issue after this update

## Update Description

### Original Problem
During security testing, it was discovered that attackers could upload files to the repository root directory without authentication.

### Problem Assessment
After further analysis, the impact of this issue is limited:
- Upload functionality is disabled by default (`UploadEnabled: false`)
- Even if enabled, attackers cannot access protected directories
- Cannot overwrite system files (like .keys)
- Root directory upload itself does not pose a direct security threat

### Solution
Converted this issue to a configuration option, allowing users to decide whether to allow root directory uploads.

## New Configuration

### AllowRootUpload (Boolean)
- **Function**: Whether to allow uploads to root directory
- **Default**: `false`
- **Description**: When enabled, allows uploading files to repository root directory

### Configuration Example
```json
{
  "UploadEnabled": true,
  "AllowRootUpload": false
}
```

## File Changes

### Modified Files
- `Models/Config.cs` - Add `AllowRootUpload` configuration option
- `Controllers/UploadController.cs` - Use configuration option to control root directory upload
- `help.txt` - Add configuration documentation

## Usage Recommendations

### Disable Root Directory Upload (Default)
- Keep directory structure clean
- Prevent accidental file pollution
- Recommend keeping disabled in production environments

### Enable Root Directory Upload
- Suitable for scenarios requiring direct upload to root directory
- Requires `UploadEnabled` to be enabled simultaneously
- Recommend using with other security measures

## Update Date
March 13, 2026

## Notes
After assessment, this issue does not constitute a high-risk vulnerability, but for better security practices, root directory upload is disabled by default.
