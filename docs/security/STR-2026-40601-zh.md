## 3. 安全响应头详解

### 3.1 X-Frame-Options

**作用**: 防止页面被嵌入到 iframe 中，避免点击劫持攻击。

**配置值**: `SAMEORIGIN`

**说明**: 只允许同源页面将此页面嵌入 iframe。

### 3.2 X-Content-Type-Options

**作用**: 阻止浏览器进行 MIME 类型嗅探，防止恶意文件被执行。

**配置值**: `nosniff`

**说明**: 浏览器将严格遵守服务器声明的 Content-Type。

### 3.3 Content-Security-Policy (CSP)

**作用**: 控制页面可以加载哪些资源，有效防止 XSS 攻击。

**配置策略**:

#### 普通页面
```
default-src 'self';
script-src 'self';
style-src 'self' 'unsafe-inline';
img-src 'self' data: blob:;
font-src 'self' data:;
media-src 'self' blob:;
connect-src 'self';
frame-ancestors 'self';
base-uri 'self';
form-action 'self';
object-src 'none';
```

#### 管理页面 (/admin)
```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self';
connect-src 'self' wss:;
frame-ancestors 'self';
base-uri 'self';
form-action 'self';
```

#### Swagger 文档 (/swagger)
```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self' data:;
connect-src 'self';
frame-ancestors 'self';
```

### 3.4 Strict-Transport-Security (HSTS)

**作用**: 强制浏览器使用 HTTPS 连接，防止中间人攻击。

**配置值**: `max-age=31536000; includeSubDomains`

**说明**: 
- 有效期 1 年（31536000 秒）
- 包含所有子域名
- 仅在 HTTPS 连接时添加

### 3.5 X-XSS-Protection

**作用**: 启用浏览器内置的 XSS 过滤器（已弃用但仍建议配置）。

**配置值**: `1; mode=block`

**说明**: 检测到 XSS 攻击时完全阻止页面加载。

### 3.6 Referrer-Policy

**作用**: 控制 Referer 头的发送行为，保护用户隐私。

**配置值**: `strict-origin-when-cross-origin`

**说明**: 
- 同源请求发送完整 Referer
- 跨域请求只发送源（不含路径）
- 降级时不发送 Referer

### 3.7 Permissions-Policy

**作用**: 控制浏览器功能的访问权限。

**配置值**: `geolocation=(), microphone=(), camera=()`

**说明**: 禁用地理位置、麦克风、摄像头权限。

## 4. 实现细节

### 4.1 中间件实现

新增 `SecurityHeadersMiddleware` 中间件，在响应发送前自动添加安全响应头。

```csharp
public class SecurityHeadersMiddleware
{
    public async Task InvokeAsync(HttpContext context)
    {
        context.Response.OnStarting(() =>
        {
            // 添加安全响应头
            // ...
            return Task.CompletedTask;
        });
        
        await _next(context);
    }
}
```

### 4.2 中间件注册顺序

安全响应头中间件应最先注册，确保所有响应都包含安全头：

```csharp
// 1. 安全响应头（最先处理）
app.UseSecurityHeaders();

// 2. IP过滤
app.UseIPBlocking();

// 3. 速率限制
app.UseRateLimiting();

// 4. 子目录路由
app.UseSubdirectoryRouting();
```

## 5. 安全建议

### 5.1 CSP 注意事项

- `'unsafe-inline'` 用于管理页面和 Swagger，因为它们使用内联脚本/样式
- 生产环境建议将内联脚本移至外部文件，移除 `'unsafe-inline'`
- WebSocket 连接需要 `connect-src 'self' wss:`

### 5.2 HSTS 注意事项

- 首次部署时建议使用较短的 max-age（如 300），确认无问题后再延长
- 一旦浏览器缓存了 HSTS，无法在有效期内取消
- 确保证书有效且不过期

### 5.3 后续改进建议

1. **CSP 报告**: 配置 `report-uri` 或 `report-to` 收集 CSP 违规报告
2. **证书透明度**: 添加 `Expect-CT` 响应头
3. **功能策略**: 根据实际需求调整 Permissions-Policy
