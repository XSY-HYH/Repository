# HRVU-202631205 - 配置更新

## 编号
**HRVU-202631205**

- **HRVU**: High-Risk Vulnerability Update（高危漏洞更新）
- **2026312**: 来源更新ID（2026年3月12日更新）
- **05**: 该更新后的第5个问题

## 更新说明

### 原始问题
在安全测试中发现，攻击者可以在无认证情况下上传文件到仓库根目录。

### 问题评估
经进一步分析，该问题影响有限：
- 上传功能默认禁用（`UploadEnabled: false`）
- 即使启用，攻击者也无法访问受保护目录
- 无法覆盖系统文件（如 .keys）
- 根目录上传本身不会造成直接安全威胁

### 解决方案
将此问题转为配置选项，由用户自行决定是否允许根目录上传。

## 新增配置

### AllowRootUpload (布尔值)
- **功能**: 是否允许上传到根目录
- **默认值**: `false`
- **说明**: 启用后允许上传文件到仓库根目录

### 配置示例
```json
{
  "UploadEnabled": true,
  "AllowRootUpload": false
}
```

## 文件变更

### 修改文件
- `Models/Config.cs` - 添加 `AllowRootUpload` 配置项
- `Controllers/UploadController.cs` - 使用配置项控制根目录上传
- `help.txt` - 添加配置说明

## 使用建议

### 禁用根目录上传（默认）
- 保护目录结构整洁
- 防止意外文件污染
- 建议生产环境保持禁用

### 启用根目录上传
- 适用于需要直接上传到根目录的场景
- 需要同时启用 `UploadEnabled`
- 建议配合其他安全措施使用
