# HRVU-20260002 - 日志接口敏感信息泄露

## 编号
**HRVU-20260002**

- **HRVU**: High-Risk Vulnerability Update（高危漏洞更新）
- **202600**: 来源更新ID（未知来源/主动发现）
- **02**: 该更新后的第2个问题

## 漏洞概述

### 发现时间
2026年4月6日

### 漏洞类型
信息泄露 - 敏感日志公开访问

### 风险等级
**高危**

### 影响范围
`/api/logs` 和 `/api/logs/{logFile}` 接口

## 漏洞详情

### 问题描述
服务器的日志接口 `/api/logs` 和 `/api/logs/{logFile}` 对所有用户公开，无需认证即可访问。日志文件可能包含敏感信息，如：

- 客户端 IP 地址
- 访问路径和请求参数
- 管理员操作记录
- 系统配置信息
- 错误堆栈信息
- 潜在的敏感文件路径

### 攻击场景

#### 1. 信息收集
```
攻击者访问:
GET /api/logs

获取所有日志文件列表，了解系统运行情况
```

#### 2. 敏感信息获取
```
攻击者访问:
GET /api/logs/2026-04-06.log

获取详细日志内容，包括:
- 管理员登录时间和 IP
- 文件访问记录
- 系统错误信息
- 配置变更记录
```

#### 3. 用户行为追踪
```
通过分析日志，攻击者可以:
- 追踪特定用户的访问模式
- 发现隐藏的目录结构
- 识别系统弱点
```

### 潜在危害
- 隐私泄露：用户 IP 和访问记录暴露
- 系统信息泄露：为进一步攻击提供情报
- 管理员信息暴露：可能被用于针对性攻击
- 业务信息泄露：文件访问模式可能暴露业务逻辑

## 修复方案

### 实现方式
1. 移除公开的 `/api/logs` 接口
2. 将日志功能移至管理面板 WebSocket
3. 要求管理员认证才能访问

### 接口变更

#### 修复前
```
GET /api/logs              → 返回日志文件列表（公开）
GET /api/logs/{logFile}    → 返回日志文件内容（公开）
```

#### 修复后
```
GET /api/logs              → 404 Not Found
GET /api/logs/{logFile}    → 404 Not Found
GET /admin/api/logs        → 404 Not Found（已移除）
WebSocket 管理面板         → get_logs / get_log 操作（需管理员认证）
```

### 访问控制
- 日志通过 WebSocket 管理面板访问
- 需要 `AdminEnabled = true`
- 需要通过 CHAP 认证

## 验证方法

### 测试公开接口（应返回 404）
```bash
curl -I https://your-server:port/api/logs
# 预期: HTTP/2 404

curl -I https://your-server:port/api/logs/2026-04-06.log
# 预期: HTTP/2 404

curl -I https://your-server:port/admin/api/logs
# 预期: HTTP/2 404
```

## 文件变更

### 修改文件
- `Controllers/SharedController.cs` - 移除公开日志接口
- `Controllers/AdminController.cs` - 添加管理日志接口

## 安全建议

### 立即行动
1. 确保已部署修复版本
2. 检查日志中是否有异常访问记录
3. 评估已泄露信息的风险

### 长期改进
1. 日志脱敏：移除或加密敏感信息
2. 日志轮转：定期清理旧日志
3. 访问审计：记录日志访问行为
4. 日志分级：分离敏感和非敏感日志
