# HRVU-202631204 - High-Risk Vulnerability Fix

## Vulnerability Number
**HRVU-202631204**

- **HRVU**: High-Risk Vulnerability Update
- **2026312**: Source Update ID (March 12, 2026 update)
- **04**: Fourth vulnerability after this update

## Vulnerability Description

### Vulnerability Type
Path Traversal / Sensitive Information Leak

### Vulnerability Details
Although the `.keys` directory is filtered from directory listings, attackers can directly access the directory and its contents by manually constructing URLs.

### Affected Paths
- `https://example.com/.keys`
- `https://example.com/.keys/server_private.pem`
- `https://example.com/.keys/server_public.pem`

### Risk Level
**High**

### Impact Scope
- Leak server RSA private key
- Leak server public key
- May lead to Secure authentication mechanism being compromised
- Attackers can forge server identity

## Vulnerability Cause

In previous fixes, the `.keys` directory display was only filtered in `DirectoryController`'s `ProcessDirectoriesAsync` method, but access control was not added at the following entry points:

1. `RepositoryController.Index` - Page access
2. `DirectoryController.GetFiles` - API directory listing
3. `FileController.PreviewFile` - File preview
4. `FileController.DownloadFile` - File download
5. `UploadController.UploadFile` - File upload

## Fix Solution

### 1. Add System Path Check Method
Added static method `IsSystemPath` in `ProtectionService` to check if path contains system directories:

```csharp
public static bool IsSystemPath(string path)
{
    if (string.IsNullOrEmpty(path))
        return false;
    
    var normalizedPath = path.Replace('\\', '/').Trim('/');
    var segments = normalizedPath.Split('/');
    
    var systemDirs = new[] { ".keys" };
    
    foreach (var segment in segments)
    {
        if (systemDirs.Any(sd => string.Equals(segment, sd, StringComparison.OrdinalIgnoreCase)))
        {
            return true;
        }
    }
    
    return false;
}
```

### 2. Add Checks at All Entry Points

| Controller | Method | Check Location |
|--------|------|----------|
| RepositoryController | Index | After path validation |
| DirectoryController | GetFiles | After path validation |
| FileController | PreviewFile | After filename check |
| FileController | DownloadFile | After filename check |
| UploadController | UploadFile | After path validation |

### 3. Unified 404 Response
All system path access requests uniformly return `404 Not Found`, not exposing whether the path exists.

5. Download request `/api/download/.keys/server_private.pem` - Should return 404
6. Upload to `/.keys/` - Should return 404

All test cases return `404 Not Found`, not leaking any system directory information.
