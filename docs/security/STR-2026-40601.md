## 3. Security Response Headers Details

### 3.1 X-Frame-Options

**Purpose**: Prevents the page from being embedded in an iframe, avoiding clickjacking attacks.

**Value**: `SAMEORIGIN`

**Description**: Only allows same-origin pages to embed this page in an iframe.

### 3.2 X-Content-Type-Options

**Purpose**: Prevents browsers from MIME type sniffing, preventing malicious files from being executed.

**Value**: `nosniff`

**Description**: Browsers will strictly follow the server-declared Content-Type.

### 3.3 Content-Security-Policy (CSP)

**Purpose**: Controls what resources a page can load, effectively preventing XSS attacks.

**Configuration Policies**:

#### Normal Pages
```
default-src 'self';
script-src 'self';
style-src 'self' 'unsafe-inline';
img-src 'self' data: blob:;
font-src 'self' data:;
media-src 'self' blob:;
connect-src 'self';
frame-ancestors 'self';
base-uri 'self';
form-action 'self';
object-src 'none';
```

#### Admin Page (/admin)
```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self';
connect-src 'self' wss:;
frame-ancestors 'self';
base-uri 'self';
form-action 'self';
```

#### Swagger Documentation (/swagger)
```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self' data:;
connect-src 'self';
frame-ancestors 'self';
```

### 3.4 Strict-Transport-Security (HSTS)

**Purpose**: Forces browsers to use HTTPS connections, preventing man-in-the-middle attacks.

**Value**: `max-age=31536000; includeSubDomains`

**Description**: 
- Valid for 1 year (31536000 seconds)
- Includes all subdomains
- Only added on HTTPS connections

### 3.5 X-XSS-Protection

**Purpose**: Enables browser's built-in XSS filter (deprecated but still recommended to configure).

**Value**: `1; mode=block`

**Description**: Completely blocks page loading when XSS attack is detected.

### 3.6 Referrer-Policy

**Purpose**: Controls the sending behavior of the Referer header, protecting user privacy.

**Value**: `strict-origin-when-cross-origin`

**Description**: 
- Same-origin requests send full Referer
- Cross-origin requests only send origin (without path)
- No Referer sent on downgrade

### 3.7 Permissions-Policy

**Purpose**: Controls access permissions for browser features.

**Value**: `geolocation=(), microphone=(), camera=()`

**Description**: Disables geolocation, microphone, and camera permissions.

## 5. Security Recommendations

### 5.1 CSP Considerations

- `'unsafe-inline'` is used for admin pages and Swagger because they use inline scripts/styles
- Production environments should move inline scripts to external files and remove `'unsafe-inline'`
- WebSocket connections require `connect-src 'self' wss:`

### 5.2 HSTS Considerations

- Recommend using shorter max-age (e.g., 300) for first deployment, then extend after confirming no issues
- Once browser caches HSTS, it cannot be cancelled within the validity period
- Ensure certificate is valid and not expired

### 5.3 Future Improvement Recommendations

1. **CSP Reporting**: Configure `report-uri` or `report-to` to collect CSP violation reports
2. **Certificate Transparency**: Add `Expect-CT` response header
3. **Feature Policy**: Adjust Permissions-Policy according to actual needs
