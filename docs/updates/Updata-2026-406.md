# Changelog - April 6, 2026

## New Features

### 1. Admin Panel (Major Update)
- Added admin backend management interface at `/admin`
- CHAP (Challenge-Handshake Authentication Protocol) encrypted authentication
- WebSocket real-time communication with support for:
  - Directory browsing and navigation
  - Move files/directories to recycle bin
  - Move/rename files/directories
  - Create new directories
  - **Upload files to current directory**
- AES-256-CBC encryption for all communication data
- Session management and key rotation

### 2. Admin Configuration
- `AdminEnabled` - Enable/disable admin features
- `AdminUsername` - Admin username
- `AdminPassword` - Admin password

### 3. Admin Connection Manager
- Added `AdminConnectionManager` service
- Tracks all active admin WebSocket connections
- Supports Ctrl+C graceful shutdown:
  - Forcefully terminates all WebSocket connections
  - Immediately stops the application

### 4. Startup Port Check
- Checks if configured port is in use before startup
- Supports IPv4 and IPv6 port checking
- Supports dual-stack mode (`::` listens on both IPv4/IPv6)
- Shows red error message when port is occupied
- Waits 5 seconds before exiting

## Improvements

### 1. File Deletion Changed to Recycle Bin
- Admin panel file/directory deletion no longer permanently deletes
- Changed to move to Windows Recycle Bin
- Frontend button text changed from "Delete" to "Recycle Bin"
- Confirmation prompt updated accordingly

### 2. Access Log Optimization
- API endpoint access no longer counted in visit counter
- Does not trigger Windows notification
- Excluded paths:
  - `/api/*` - API endpoints
  - `/swagger/*` - Swagger documentation
  - `/download/*` - Download endpoints
  - `/admin/ws` - WebSocket endpoint
  - `*/ws` - Other WebSocket endpoints
- API access is still logged (marked as "API access")

## Bug Fixes

### 1. Routing Issue
- Fixed `/admin` path incorrectly routed to repository directory
- `SubdirectoryRoutingMiddleware` excludes `/admin` path

### 2. WebSocket Connection Issues
- Fixed HTTP method mismatch (CONNECT vs GET)
- Fixed JSON serialization issue (PascalCase vs camelCase)
- Fixed infinite loop issue (duplicate message handling)

### 3. Path Handling Issues
- Fixed path normalization issues
- Correctly handles Windows file system paths
- Removed leading slashes to avoid absolute path issues

### 4. WebSocket Close Exception
- Fixed exception when server tries to close after client already closed
- Added WebSocket state checking
- Safe close method catches exceptions to prevent crashes

### 5. Missing HTTP Security Response Headers
- Fixed missing critical security response headers
- Added 7 security response headers for protection
- Prevents clickjacking, XSS, MIME confusion attacks
- See HRVU-20260001 vulnerability report

### 6. Logs API Security Hardening
- Disabled public `/api/logs` endpoint (may leak sensitive information)
- Moved logs endpoint to admin panel `/admin/api/logs`
- Requires admin authentication to access log files

## Technical Details

### CHAP Authentication Flow
1. Client generates random key (SHA256)
2. Sends encrypted username
3. Server validates and returns session ID and new key
4. Subsequent communication uses session key encryption

### Ctrl+C Handling Flow
1. User presses Ctrl+C
2. Cancel default termination behavior
3. Forcefully terminate all WebSocket connections
4. Immediately stop the application

## Upgrade Notes

1. After updating, admin account configuration is required
2. Set in `Config.json`:
   - `AdminEnabled`: true
   - `AdminUsername`: admin username
   - `AdminPassword`: admin password
3. Visit `https://your-server:port/admin` to access admin panel
4. Admin panel delete operation behavior changed (move to recycle bin instead of permanent deletion)
5. If port is occupied at startup, it will prompt and exit

## Security Notes

- Admin panel uses CHAP protocol, password is not transmitted over network
- All communication uses AES-256-CBC encryption
- Session key rotates after each operation
- Recommend using HTTPS to access admin panel
