# Changelog - March 12, 2026

## New Features

### 1. Windows Notification System
- Added `Notification` configuration option
- Support for Windows native Toast notifications
- Real-time alerts for access events, warnings, errors
- Automatically disabled on non-Windows systems

### 2. Secure Authentication Client Example
- Added `secure_client.py` parameter-driven client
- Command-line argument support for server URL, client ID, token, etc.
- Complete Secure authentication flow implementation

### 3. Project Documentation
- Added `README.md` project description document
- Includes core features, project structure, API endpoints, configuration, etc.

## Security Improvements

### 1. Protected Directory Hiding
- Protected directories no longer appear in directory listings
- Direct access to protected directories returns 404, not exposing directory existence
- Prevents attackers from probing protected directories through page behavior

### 2. Automatic System Directory Blocking
- `.keys` directory automatically hidden to prevent key leakage
- No manual blacklist configuration needed

### 3. Path Normalization Fix
- Fixed path comparison format inconsistency
- Ensured `IsPathProtected` correctly identifies protected paths

## Bug Fixes

### 1. Timestamp Handling
- Fixed Secure authentication timestamp byte order issue
- Server and client unified to use big-endian

### 2. Secure Authentication Flow
- Fixed Secure Handler not created after client registration
- Added `RefreshSecureHandlerForClient` method
